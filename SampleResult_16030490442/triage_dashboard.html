Key Findings
  • 1DOMINANT CLUSTER — IP Fuse/Configuration (6/13 tickets): The most systemic issue is the incomplete BISR (Built-In Self Repair) fuse controller RTL in the ww13a model. Tickets 14027643689, 14027663847, 14027663856, 14027664554, and 14027686796 are a direct consequence of this single RTL gap — all were rejected by DFT with force files provided for emulation to proceed. This is a hw.arch/hw.bug concern: BISR fuse IP controller FSMs (mBISR_rstw_* and fBISR_rstw_*) across punit, d2d, msm_aggr, iommu, cxlcm, fblp, hiop, scatop, ubr, scrs, scrsp2p, cms, fws, dsa, hap, iax, and LCP_ctrl are all blocking HWRS phases 3 and 4.
  • 2SYSTEMIC ROOT CAUSE — Port ID / Strap Tieoff Misconfiguration: Ticket 14027339278 (port IDs tied to zero for HWRS, S3M, DFX Aggregator) directly caused S3M sideband packet drops (14027482819, potentially 14027492988) as the port_id tieoff errors prevented correct routing through the fabric. The fix in manual_connectivity.py and config_utils.py resolved the strap automation defects for subsequent model builds.
  • 3HWRS LOGIC BUG — Reset/Power Sequencing (hw.bug candidate): Ticket 14027683356 exposes an RTL logic error in the HWRS IP where imh2cbb_hwsync_req_out is asserted on D2D during Phase 3 (PHASE_3_CBB_DIE_READY) even when SKU_FEATURE_DWORD1.imh2cbb_hwsync_connected = 0x0 (CBBs absent in IOH SKU). This is a hw.bug — the HWRS RTL condition does not correctly suppress cross-die sync signals for disabled interfaces. Fixed via ww16 HWRS IP uprev.
  • 4Reset Domain Misconfiguration: Ticket 14027520901 identified the S3M fabric router being wired to the wrong reset domain (soc_early_infra/S0 instead of s5_crux), causing S3M sideband transactions to stall. Root cause was a commented-out line in the config automation (config_utils.py). This is a configuration/connectivity correctness issue affecting reset bringup.
  • 5Low-confidence open issue: Ticket 14027675621 (punit sideband drops in router 2167 for isa_oob, isa_uio, isa_scf_center0top) remains unresolved with no root cause established as of the ticket status. This may be a follow-on from the strap tieoff fixes or an independent routing table misconfiguration.
  • 6Tooling noise: Ticket 14027743431 is a false positive — a primecode fw_trace_parser invocation error due to a relative path mistake, with no hardware or RTL impact. Successfully resolved by using the absolute path to xtmp_trace.out.
